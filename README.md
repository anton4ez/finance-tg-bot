# Finance Analyst Bot

Бот запущен и доступен по ссылке: https://t.me/finance_managerr_bot

Данный проект официально развернут на удаленном VPS сервере и работает 24/7.

## Описание проекта
Telegram-бот для автоматизированного учета личных финансов, анализа текущих расходов и генерации аналитических отчетов. Система построена на асинхронной архитектуре и использует реляционную базу данных для надежного хранения данных.

## Функциональные возможности
* Учет расходов: Добавление транзакций через систему состояний.
* Категории: Распределение трат по настраиваемым категориям.
* Визуализация: Автоматическая генерация круговых диаграмм на основе статистики.
* Экспорт данных: Выгрузка полной истории транзакций в формате Excel (.xlsx).
* База данных: Хранение данных в PostgreSQL с использованием SQLAlchemy 2.0.

## Технологический стек
* Язык: Python 3.12+
* Фреймворк: Aiogram 3
* ORM: SQLAlchemy 2.0
* БД: PostgreSQL
* Аналитика: Pandas, Matplotlib
* Деплой: Ubuntu Server, systemd, Git

## Запуск через Docker (Рекомендуемый способ)

Проект полностью контейнеризирован. Для быстрого запуска вам понадобятся только Docker и Docker Compose.

1. Клонируйте репозиторий:
   git clone https://github.com/ВашНик/finance-tg-bot.git
   cd finance-tg-bot

2. Создайте файл .env в корне проекта.
   BOT_TOKEN=ваш_токен
   DB_HOST=db
   DB_PORT=5432
   DB_USER=postgres
   DB_PASSWORD=ваш_пароль
   DB_NAME=finance_bot

3. Соберите и запустите контейнеры в фоновом режиме:
   docker-compose up -d --build

Система автоматически скачает образ PostgreSQL, соберет образ бота с нужной версией Python, установит зависимости и свяжет их в единую виртуальную сеть.

## Структура проекта
* bot/ — Обработка команд, фильтры и клавиатуры.
* database/ — Модели данных и логика взаимодействия с PostgreSQL.
* analytics/ — Модули формирования графиков и Excel-отчетов.
* main.py — Точка входа в приложение.
* .env — Конфигурационный файл (исключен из репозитория).

## Развертывание
Проект работает в фоновом режиме на удаленном сервере (VPS) под управлением службы systemd, что обеспечивает автоматический перезапуск и доступность 24/7.
